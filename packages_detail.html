State Management & Data Flow

React State (Client Component)
useState hooks:
• customizations: { dayIndex: 'luxury' | 'standard' }
• isCustomizing: boolean (toggle mode)
• recentlyViewed: array (from localStorage)

Derived values:
• hasUpgrades: boolean (any customizations selected)
• upgradeCount: number (luxury day count)
• dynamicPrice: $basePrice + (upgradeCount × $250)
Effects & Animations (GSAP)
useGSAP hook manages:
• Hero image parallax scrolling
• Staggered reveal animations on page load
• Section header fade-ins on scroll
• Individual itinerary card reveals
• ScrollTrigger refresh on customization state change

Event handlers:
• toggleCustomization(idx, type): Updates customizations state
• setIsCustomizing(boolean): Toggles edit mode
Data Source
From data.json (server):
• Package details (pkg object): id, slug, title, category, location, price, duration, image, description, itinerary
• Location details: name, country

From localStorage (client):
• recentlyViewed: array of package slugs (max 5)
• Updated on component mount
Price Calculation Logic
base_price = $1,200 (from pkg.price)
upgrade_count = Object.values(customizations).filter(v => v === 'luxury').length
final_price = base_price + (upgrade_count × $250)

Example: 2 luxury upgrades → $1,200 + (2 × $250) = $1,700
Responsive Layout
• Mobile (<768px): Single column layout. Sidebar below content. Sticky disabled.
• Tablet/Desktop (≥768px): Two-column grid (8 cols / 4 cols). Sidebar sticky at top-32.
• Hero height: 85vh (scales responsively)
• Promise strip: 4-col grid → 2-col on tablet → 1-col on mobile

User Interaction Flows

Flow 1: Browse Package Details
1. User arrives at /packages/[slug] page
2. Hero parallax animates on scroll
3. Sections fade in with GSAP reveals
4. Itinerary cards load with individual scroll triggers
5. Sidebar remains sticky during scroll

Outcome: Immersive, animated package discovery experience.
Flow 2: Customize Itinerary
1. User clicks [Personalize Journey] button
2. isCustomizing state → true (button highlight, status change to "Confirm Personalization")
3. Day cards become clickable (hover states, cursor:pointer)
4. User clicks day cards to toggle between "standard" & "luxury"
5. Clicked card updates: background color, shows upgrade badge, displays perks
6. Sidebar updates:
• dynamicPrice recalculates in real-time
• Custom Summary section appears (shows each upgraded day + cost)
• Configuration field changes from "Standard" → "Bespoke"
7. CTA button text changes: "Begin Reservation" → "Request Bespoke Quote"

Outcome: User creates personalized itinerary with real-time pricing feedback.
Flow 3: Proceed to Booking
1. User review final price & customizations
2. Clicks primary CTA ("Begin Reservation" or "Request Bespoke Quote")
3. Navigate to /booking with package slug + customizations in URL params (optional)
4. Alternatively, user clicks "Speak to an Expert" → Initiates phone call or chat

Outcome: Lead capture and booking initiation.
Flow 4: Recently Viewed Tracking
1. User lands on package detail page
2. useEffect hook on mount: reads localStorage["recentlyViewed"]
3. Filters to exclude current package slug
4. Renders up to 3 recent packages in grid
5. Updates localStorage with new visit (max 5 total)
6. User clicks recent card → navigates to that package

Outcome: Personalized "continue exploring" experience across sessions.
