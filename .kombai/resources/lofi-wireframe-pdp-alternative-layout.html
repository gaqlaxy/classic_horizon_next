Design Overview: Alternative Layout

Key improvements over original 2-column grid design

Layout Strategy
Desktop (1024px+): Two-column grid with floating sticky sidebar. Content flows on left, sidebar stays in viewport.
Tablet (769-1023px): Single column with sticky sidebar (slight offset). Better tablet experience.
Mobile (<768px): Full single-column stack. Sidebar becomes inline card. No sticky positioning. Better touch interactions.

CSS Approach: Uses native `position: sticky` instead of JavaScript scroll tracking. More performant, fewer mobile bugs.
Component Breakdown
Section Current Design Alternative Design Benefit
Hero 680px (85vh) 500px (fixed) Faster scroll to content. Better mobile UX.
Promise Strip Fixed 4-col grid Auto-responsive grid Automatically adapts to screen size. No hardcoded breakpoints.
Inclusions Sidebar callout in content Separate grid cards More prominent. Easier to scan. Better visual balance.
Itinerary Stacked full-width rows Responsive card grid (3→1 cols) Better visual separation. Cleaner on desktop. Responsive without media queries.
Sidebar 2-col grid column with margin Floating sticky card Stays visible during scroll. Better CTA visibility. Mobile-optimized.
Navigation None (single page) Optional tabs Content organization. Future-proof for reviews section.
Responsive Behavior
Desktop (1024px+):
• Content + Sidebar in 2-col grid (1fr 320px)
• Sidebar sticky at top: 20px
• Height: fit-content (grows with sidebar content)
• Itinerary cards: 3 per row
• Inclusions cards: 4 per row (auto-fit)

Tablet (769-1023px):
• Still 2-col but sidebar less prominent
• Itinerary cards: 2 per row
• Inclusions cards: 2 per row

Mobile (<768px):
• Content full-width
• Sidebar below (position: relative, not sticky)
• Itinerary cards: 1 per row
• Inclusions cards: 1 per row
• Promise strip: 2 cols (better on mobile)
• Tabs: Full width with horizontal scroll if needed

Key Features & Interactions
1. Floating Sticky Sidebar: Remains in viewport during scroll. Disables on mobile for better touch experience.

2. Responsive Card Grids: Uses `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))`. Automatically adjusts columns based on available space. No manual breakpoints needed for itinerary.

3. Tab Navigation: Optional section tabs (Overview/Details/Itinerary/Reviews). Allows content organization without page reload. Easy to add future sections like reviews/photos.

4. Dynamic Pricing: Same state management as original. Sidebar updates in real-time as user selects luxury upgrades. Price breakdown shows itemized costs.

5. GSAP Animations: Hero parallax, section reveals, and scroll triggers remain unchanged. ScrollTrigger refresh handles layout shifts during customization.

6. Inclusions as Grid: Instead of sidebar callout, inclusions are prominent cards. Each card has icon space (future) for visual appeal.
